USD Layout Pre-Production Pipeline

Work on our animated short Red Hat Ringo began in January of 2023, and my main task since the project started has been creating a workflow for layout that uses Universal Scene Description (USD), a recently released software platform developed by Pixar that manages 3D animation data efficiently, to handle managing layouts and set dressing. I was tasked with designing a way for layout artists to assemble a set using USD rather than the old assembly workflow, and I did this by automating essential parts of the USD pipeline and creating a tool artists could use to easily manage the USD data in their layout.

Overview of Pipeline Steps

  1. The modeling team publishes their work which creates the supporting assembly files. Creating the supporting assembly files exports the proxy (and render, if it exists) versions of the model to USD and stores them on the network.
  2. When a layout artist first opens an assembly asset, a base USD file named after the asset is created and to be imported into their Maya scene. When the base USD is created:
    1. The USD kind is set to assembly. This is done so that the layout can be imported into another layout in the future if needed.
    2. A prim named ‘contents’ is created and set to be the default prim for the USD. All assets in the future are automatically imported into the contents prim.
  3. The layout artist uses the USD Manager tool to add and remove assets to the USD, then they can move them around as needed.




    1. The tool queries ShotGrid to gather project data and assemble a list of all a ssets in the project, with icons that indicate whether layout is able to pull in the asset.
      1. Green icons indicate that the asset's USD file exists on the network so the artist can therefore import the asset into their layout.
      2. Red icons indicate that the asset cannot be imported.
      3. Yellow icons indicate an assembly asset.
    2. The artist can check the assets they want to import into the layout (Import Checked Assets).
      1. When they import an asset, it creates a prim with a USD reference.
    3. The artist can check the assets they want to remove from the layout (Remove Checked Assets).
    4. The artist can choose to reload the USD if they would like to reload any changes made to any USD references while they were working with it (Reload All Assets).
  4. From here, the USD can be pulled into Animation’s shots in Maya as well as into Lighting’s shots in Unreal (our pipeline has departments working between Unreal and Maya). This same tool shown above also exists in the Unreal workflow since artists are able to open and edit assets in both Maya and Unreal in our pipeline.


Layers

The above image shows the USD layer editor and how sublayers can be used in the USD workflow. In the outliner, you can see the prims that were imported into the base USD, as well as the base prim 'contents' that holds the entire layout. Whenever an artist imports an asset into the USD, it automatically makes those changes in the base layer, which contains the initial layout. If an artist makes any changes to the layout that is not included in the initial layout, they choose a different layer based on their department.

About Bad Timing

I joined Bad Timing for the second semester of its production, so the main focus for the semester was surfacing, rigging, animation, and lighting. Jumping straight into pipeline management with only a semester of previous tools development experience was very challenging; my teammate and I were given the code for the main modules, and we spent a majority of our time for the first semester learning and experimenting with the pipeline. We mostly focused on troubleshooting problems and bugs that artists ran into, especially for the last few weeks to get the final short ready for the premiere.

Beyond just troubleshooting, my teammate and I also worked on a way for surfacing artists to work locally. One of the artists' biggest recurring problems is our network drive due to how slow it is to access files work on pipe. As a result, previous technical directors worked on ways to download files from the network so artists could work on them locally then have them version back up to the network when they finish working. This workflow is much simpler for a discipline like modeling, but with surfacing it was a little more involved because we needed to take into account file dependencies and handle the workflow accordingly.
